# 捡尸监控 — 方案设计

> 目标：监控钱包内所有"垃圾 meme"持仓，任何一个标的出现异动（暴涨/暴跌/新增 token）时，第一时间推送 Telegram 通知，不错过行情。

---

## 一、所需材料清单

### 1. 你需要提供的（必须）

| # | 材料 | 说明 | 获取方式 |
|---|------|------|----------|
| 1 | **Solana 钱包地址** | 你要监控的捡尸钱包公钥 | 你自己的钱包，已有 |
| 2 | **Telegram Bot Token** | 用来给你推消息的机器人 | [@BotFather](https://t.me/BotFather) → `/newbot` |
| 3 | **Telegram Chat ID** | 你的个人 Chat ID | [@userinfobot](https://t.me/userinfobot) 发条消息即可获取 |

### 2. 免费 API（不需要你申请）

| API | 用途 | 费用 | 限制 |
|-----|------|------|------|
| **Solana 公共 RPC** (`api.mainnet-beta.solana.com`) | 读取钱包所有 SPL Token 持仓 | 免费 | 有 rate limit，但 5 分钟跑一次够用 |
| **DexScreener API** (`api.dexscreener.com`) | 查询每个 token 的价格、5m/1h/6h/24h 涨跌幅、流动性、市值 | 免费，无需 Key | 每次最多查 30 个 token，我们分批请求 |

### 3. 基础设施（免费）

| 组件 | 方案 | 说明 |
|------|------|------|
| **定时调度** | GitHub Actions Cron | 每 5 分钟触发，免费额度 2000 分钟/月（私有仓库）或无限（公开仓库） |
| **状态持久化** | `state.json` commit 回仓库 | 记录上次已知 token 列表 + 报警冷却时间 |
| **通知渠道** | Telegram Bot API | 免费，无消息量限制 |

### 4. GitHub Secrets 配置（安全起见）

在仓库 → Settings → Secrets and variables → Actions 中添加：

| Secret 名称 | 值 |
|---|---|
| `WALLET_ADDRESS` | 你的钱包地址 |
| `TELEGRAM_BOT_TOKEN` | Bot Token（别直接写代码里！） |
| `TELEGRAM_CHAT_ID` | 你的 Chat ID |

## 二、监控逻辑设计

### 信号规则

| 类型 | 触发条件 | 说明 |
|------|----------|------|
| 🚀 暴涨 | 5分钟涨 ≥ 30% | 短时间拉盘，可能是起飞 |
| 🚀 暴涨 | 1小时涨 ≥ 50% | 持续走强 |
| 🚀 暴涨 | 6小时涨 ≥ 100% | 翻倍确认 |
| 🔻 暴跌 | 5分钟跌 ≤ -30% | Rug/砸盘 |
| 🔻 暴跌 | 1小时跌 ≤ -50% | 流动性抽干 |
| 🆕 新增 | 钱包出现之前没有的 token | 提醒你关注新仓位 |

### 防刷屏机制

- **冷却时间**：同一个 token 报警后 60 分钟内不重复推送
- **粉尘过滤**：持仓价值 < $1 的 token 忽略
- **首次运行**：不推新增 token 提醒（避免一次几十条）

### 推送内容

每条 Telegram 消息包含：
- 币名 + 合约地址
- 当前价格 + 持仓数量 + 持仓价值
- 流动性 + 市值
- 5m / 1h / 6h / 24h 涨跌幅
- 一键跳转链接：GMGN / DexScreener / Photon
